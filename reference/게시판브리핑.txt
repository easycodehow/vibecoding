---------------------------------------
1. 게시판 기능 범위
---------------------------------------
- CRUD (등록, 조회, 수정, 삭제) 구현

- 댓글 기능: 작성 / 수정 / 삭제

- 파일 첨부: 이미지 파일 1개만 가능 (PDF 등 다른 형식 첨부 불가)
  → 이미지 업로드 시 미리보기 제공

- 검색 기능: 제목 기준 검색

- 페이지네이션 방식: **페이지 번호를 5개 단위로 노출**
  (예: 처음엔 1, 2, 3, 4, 5 표시 → "다음" 클릭 시 6, 7, 8, 9, 10 표시)

- 조회수 집계 방식: 단순 카운트*(같은 사용자가 새로고침해도 중복 방지 없이 +1)

---------------------------------------
2. 게시판에 보여질 내용
---------------------------------------
- 게시글 제목
- 작성자 (아이디)
- 작성일시
- 조회수

---------------------------------------
3. 로그인 / 권한 시스템
---------------------------------------
- 로그인 방식: 이메일 + 비밀번호(회원가입 시 이메일 사용, "아이디" 로그인 아님)

- 비로그인 사용자: 읽기만 가능 (글쓰기·댓글·수정·삭제 불가)

- 로그인 사용자: 본인이 작성한 글/댓글만 수정·삭제 가능

- 관리자 권한:
  - 부여 방식: 테이블에 역활 컬럼을 만들어서 관리자역활지정
  - 관리자가 할 수 있는 것: 모든 게시글 삭제 가능
  - 할 수 없는 것: 회원 관리 기능 없음 (계정 정지, 권한 변경 등 불필요)

---------------------------------------
 4. 게시판 디자인
---------------------------------------
- 목록 화면 레이아웃: 테이블형
- 상세보기 화면 구조: [레퍼런스 이미지 참고]
- 모바일 반응형 필요

---------------------------------------
5. 데이터베이스 구조
---------------------------------------
< posts 테이블 >
- 글번호 (자동 생성)
- 제목
- 내용
- 작성자
- 작성날짜
- 조회수
- 이미지 URL (1개) 

< comments 테이블 >
- 댓글번호
- 원글 번호 (posts 참조)
- 작성자
- 내용
- 작성날짜

< profiles 테이블>
- 사용자 ID (auth 연동)
- role (일반 사용자 / 관리자 구분)
