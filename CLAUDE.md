# 바이브코딩 프로젝트 가이드

# 프로젝트 개요
- 주제 : 바이브코딩
- 목적 : 바이브코딩의 정의, 특징, 기대효과 등에 대한 개론적인 설명으로 이해를 돕고자 함
- 타깃 사용자 : 바이브코딩이 무엇인지 알고싶어하는 모든 사람
- 배포: Vercel (vibecoding-pi-three.vercel.app)
- 폴더명: vibecoding

## 기술 스택
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla)
- **Backend/Database**: Supabase
- **배포**: Vercel
- **버전관리**: GitHub

## 코딩 규칙

### HTML
- 시맨틱 태그 우선 사용 (header, nav, main, article, section, footer)
- 주석은 한국어로 작성
- 들여쓰기는 2칸

### CSS
- 클래스명은 명확한 의미 전달
- Vanilla CSS 사용 (프레임워크 사용 금지)
- 주석으로 섹션 구분

### JavaScript
- ES6+ 문법 사용
- 가독성을 위해 함수는 작게 분리
- 주석은 한국어로 작성

## 작업 규칙

### 1. 승인 원칙
- "시작" 또는 "go"라는 말이 나오기 전까지는 코드를 읽고 분석만 하고, 실제로 파일을 만들거나 수정하지 않을 것.
- 승인은 **세션 전체에 한 번이 아니라, 각 단계(헤더, 슬라이더, 푸터 등)마다 매번** 받을 것.
- 한 단계가 끝나면 자동으로 다음 단계로 넘어가지 말고, 반드시 새로 "시작"/"go"를 기다릴 것.
- 작업을 시작하기 전, 다음 형식으로 계획을 먼저 설명할 것.
  > "이번 단계에서는 [무엇을] 위해 [어떤 파일들]을 [어떻게] 수정하겠습니다."

### 2. 작업 기록 규칙 (이전 작업 기억하기)
- 새 세션을 시작할 때 먼저 `PROGRESS.md` 파일이 있는지 확인하고, 있으면 내용을 읽을 것.
- 가장 마지막에 기록된 작업 내용을 요약해서 사용자에게 보고할 것.
- "다음 작업 제안"이 적혀 있으면 함께 보고할 것.
- `PROGRESS.md`가 없으면 "이전 작업 기록이 없습니다"라고 알릴 것.
- `PROGRESS.md` 맨 아래에 기록을 추가할 것 (기존 기록은 지우지 말고 누적할 것).

## 현재 작업: 게시판(Board) 기능 추가

### 게시판 이름
- 커뮤니티

### 확정된 요구사항

**기본 기능**
- 글 번호는 자동 생성
- 제목, 내용, 작성자, 작성일, 조회수 표시 (목록/상세 공통)
- 이미지 1개 업로드 가능 (이미지 파일만, 그 외 파일 첨부 없음)
- 이미지 업로드 시 미리보기 제공
- 제목 검색
- 페이지네이션: 페이지 번호 5개 단위로 노출 (1~5 표시 → 다음 클릭 시 6~10)
- 조회수: 단순 카운트 (중복 방지 로직 없음, 조회할 때마다 +1)

**댓글 기능**
- 작성 / 수정 / 삭제 모두 가능
- 본인 댓글만 수정/삭제 가능

**로그인/권한**
- 이메일 로그인/회원가입 (Supabase 기본 방식)
- 비로그인 사용자: 읽기만 가능
- 로그인 사용자: 글쓰기 가능, 본인 글/댓글만 수정·삭제 가능
- 관리자: `profiles` 테이블에 `role` 컬럼을 추가해 지정. 관리자는 모든 게시글 삭제 가능 (회원 관리 기능은 없음)
- 로그인 세션은 브라우저(탭/창)를 닫으면 유지되지 않음 (새로고침이나 페이지 이동 중에는 유지, 브라우저를 완전히 닫았다 다시 열면 로그아웃 상태)

### 기술 스택
**프론트엔드 (화면)**
- Vanilla JavaScript - 프레임워크 없는 순수 자바스크립트
- HTML, CSS

**백엔드 (서버/데이터베이스)**
- Supabase - 데이터베이스, 로그인, 파일 저장 기능 제공

**배포 (호스팅)**
- Vercel - 무료로 웹사이트 배포
